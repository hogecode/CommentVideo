## 概要

<!--

動画プレイヤーの上に、ニコニコ動画風の弾幕コメントをオーバーレイ表示する機能を実装します。
ユーザーは動画再生中にコメントが流れる様子を楽しむことができ、まるで他のユーザーと一緒に動画を視聴しているような体験を得られます。

コメントには「右から左へ流れる」「画面上部に固定」「画面下部に固定」の3種類の表示タイプがあり、文字サイズ・色も個別に設定できます。また、コメントの表示タイミングを動画に対してずらす「遅延オフセット」機能を提供します。

**目的**:

- 動画視聴体験の向上（コメントによる賑やかさ・共感の演出）
- ニコニコ動画に慣れたユーザーへの馴染みやすい UI の提供

### 機能要件

<!--

#### 動画の上にコメントを表示するフロー

1. **ページ表示**:
   - ユーザーがトップページ（`/videos`）にアクセスすると、動画プレイヤーと弾幕コメントが表示される
   - 動画は `page.tsx` で指定した URL（現状はサンプルの `blank30.mp4`）を再生する
  
2. **コメントの流れ方**:
   - コメントは再生時刻に対応したタイミングで表示される
   - `type: 'right'` → 右端から左端へ横スクロールして流れる（最も一般的な弾幕）
   - `type: 'top'` → 画面上部に固定表示される
   - `type: 'bottom'` → 画面下部に固定表示される
3. **コメントの見た目**:
   - `color` フィールドで各コメントの文字色を個別に指定できる（例: `#ffffff`, `#ffd93d`）
   - `size` フィールドで文字サイズを指定できる（`'big'` / `'medium'` / `'small'`）
4. **コメント遅延調整（オフセット機能）**:
   - ユーザーはスライダーで −30 秒〜+30 秒の範囲でコメント表示タイミングをずらせる
   - 正の値: コメントを指定秒数だけ遅らせて表示（例: +5 → 動画5秒時点で0秒のコメントが出る）
   - 負の値: コメントを指定秒数だけ先行して表示
   - 「リセット」ボタンでオフセットを 0 に戻せる
5. **エラーハンドリング**:
   - 動画 URL が空・存在しない場合、DPlayer 標準のエラー表示を使用する
   - コメントデータが空配列の場合、コメントなしで再生のみ行う

#### パフォーマンス

- DPlayer は動的インポート（`import('@/danmaku/DPlayer')`）で遅延ロードし、初期バンドルサイズを削減する
- コメントデータは `useRef` で管理し、再レンダリングなしに最新データを参照できるようにする

#### ユーザビリティ（UX）

- 遅延オフセットのスライダーはリアルタイムに反映され、シークなしで即時同期される
- スライダーのラベルは「+○秒 遅延」「○秒 先行」「遅延なし」のように人間が読みやすい形式で表示する
- コントロール UI は動画プレイヤーの直下に配置し、動画再生を妨げない

#### 保守性

- DPlayer はローカルに同梱（`src/danmaku/DPlayer/`）しているため、外部パッケージバージョンアップの影響を受けない
- `deprecated/DanmakuPlayer/` は参照用として残すが、実装では使用しない

### 技術スタック

- **フレームワーク**: Next.js（App Router）, TypeScript
- **UIライブラリ**: MUI（状態管理や汎用コンポーネントに使用）
- **動画・弾幕プレイヤー**: DPlayer（ローカル同梱: `src/danmaku/DPlayer/`）
- **スタイリング**: Tailwind CSS（ページレイアウト）、DPlayer 標準 CSS（プレイヤー UI）

### コンポーネント構成

```
src/app/page.tsx
  └── DPlayerVideo  (src/components/DPlayerVideo/index.tsx)
        └── DPlayer (src/danmaku/DPlayer/)   ← 動画再生 + 弾幕エンジン
```

#### `src/app/page.tsx`

- アプリのエントリーポイント（トップページ）
- 動画 URL とコメントデータ（`DPlayerType.Dan[]`）を `DPlayerVideo` に渡す
- 現状はハードコードされたサンプルデータを使用（将来的にはAPIから取得）
- `page.tsx` がコメントデータ・動画ソースの責務を持ち、`DPlayerVideo` は表示に専念する

**コメントデータ型**:

```typescript
// DPlayerType.Dan の各フィールド
type Dan = {
  id: string;        // コメントの一意ID
  time: number;      // 表示する動画再生時刻（秒）
  text: string;      // コメント本文
  color: string;     // 文字色（CSS カラーコード例: '#ffffff'）
  type: 'right' | 'top' | 'bottom';  // 流れ方
  size: 'big' | 'medium' | 'small';  // 文字サイズ
};
```

#### `src/components/DPlayerVideo/index.tsx`

DPlayer の React ラッパーコンポーネント。以下の責務を持つ:

1. **DPlayer の初期化・破棄**
   - `containerRef` に DPlayer を `useEffect` でマウントする
   - `src` が変わったら DPlayer を再初期化する（`return () => dp.destroy()`）
   - SSR 対策として DPlayer は動的インポート（`import('@/danmaku/DPlayer')`）する
2. **コメントデータのローカル提供（`apiBackend`）**
   - DPlayer の `apiBackend.read` をカスタム実装し、`danListRef.current` からコメントを提供する
   - `danListRef` で管理することで、`danList` が更新されても再レンダリングなしに最新データを参照できる
3. **コメント遅延オフセット機能**
   - DPlayer 初期化後に `dp.danmaku.options.time` を差し替えることで実現する
   - 差し替え後の time 関数: `() => dp.video.currentTime - delayOffsetRef.current`
   - DPlayer 内部の `frame()`（コメント判定）と `seek()`（シーク後の再同期）は両方この `options.time()` を参照するため、差し替えだけで自動的に適用される
   - `handleDelay(value)`: オフセット値を更新 → `dpRef.current.danmaku.seek()` を呼んで即時再同期

**Props**:

```typescript
interface Props {
  src?: string;              // 動画ファイルの URL（省略時は空文字）
  commentList?: Comment[];   // 初期表示する弾幕データ（省略時は空配列）
}
```

```
|  |    （弾幕コメントがオーバーレイで流れる）     |  |
|  |                                            |  |
|  +--------------------------------------------+  |
|                                                  |
|  コメント遅延調整          遅延なし  [ リセット ]   |
|  ← 30秒先行 ─────────────────────── 30秒遅延 →   |
|              ○（スライダー）                      |
+--------------------------------------------------+
```

### コメント遅延オフセットの実装原理

DPlayer の弾幕エンジン（`danmaku.frame()`）は以下のロジックでコメントを表示する:

```
// DPlayer 内部の擬似コード
if (options.time() > item.time) {
  // コメントを描画
}
```

`options.time` を `() => video.currentTime - offset` に差し替えることで:

- `offset > 0`: `options.time()` が小さくなる → コメントの出現が遅くなる（遅延）
- `offset < 0`: `options.time()` が大きくなる → コメントの出現が早まる（先行）

これにより、コメントデータを直接書き換えることなく、表示タイミングのずれを吸収できる。
